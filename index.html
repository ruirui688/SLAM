P248 手机端动态区域标注

详细标注指南 / 边界说明

标注目标

本页面用于人工判断每个样本中是否存在需要关注的动态区域或疑似动态区域，并记录模型 overlay 是否有误报、漏报。请只根据当前样本可见内容填写，不要查看 admission labels 或其他会影响独立判断的信息。

P195 仍为 BLOCKED：导出的非空标签必须经过覆盖率、冲突和独立性审计后才能使用。

先看什么

  • Raw 原图是主要依据。先用 Raw 判断画面里是否真的有动态或临时物体。
  • Probability / Soft mask / Crop 只是帮助定位。它们来自模型或裁剪，不是真值，不要因为 overlay 有颜色就直接判定为动态。
  • 如果 Raw 看不清、遮挡严重、反光明显或画面模糊，优先选择 uncertain / low，并在 reviewer_notes 写明原因，不要硬猜。

推荐标注流程

  1. 先看 Raw 原图，判断是否有明显动态区域或临时物体。
  2. 切到 Probability 和 Soft mask，看 overlay 标到哪里，只把它当作定位参考。
  3. 查看 Crop，确认局部区域是否支持你的判断。
  4. 填写 dynamic_region_present 和 dynamic_region_type。
  5. 评估 boundary_quality，并判断 false_positive_region / false_negative_region。
  6. 选择 label_confidence；有任何不确定就在 reviewer_notes 写原因。
  7. 点“保存”或直接“下一条”；全部完成后点底部“导出 CSV”，把文件发回审计。

字段怎么填

dynamic_region_present
yes：Raw 原图中能看到明确的动态区域、移动物体、临时物体或明显不稳定区域。
no：Raw 原图中看不到动态区域，overlay 只是落在墙、地面、货架等固定结构上。
uncertain：看不清、被遮挡、反光强、模糊、区域太小，无法可靠判断。
dynamic_region_type
moving_object：人、车、搬运设备、移动货物等明显会移动或临时出现的对象。
static_structure：墙、地面、柱子、货架、固定设备等静态结构；如果 overlay 主要覆盖这些，通常不是动态目标。
reflection：反光、玻璃/金属高光、镜面造成的疑似区域。
shadow：阴影、光照变化造成的疑似区域。
unknown：能看到异常区域，但无法判断具体类型。
not_applicable：dynamic_region_present 为 no，或确实没有可分类的动态区域。
boundary_quality
good：overlay 边界大体贴合目标，少量误差不影响判断。
acceptable：overlay 覆盖到目标的一部分，或边界有偏移但仍能作为参考。
poor：overlay 明显偏离目标、覆盖错误区域，或边界质量很差。
not_applicable：没有动态区域，或没有可评估的 overlay 边界。
false_positive_region
如果模型标了区域，但 Raw 原图看起来不像动态/目标区域，例如只覆盖墙、地面、固定结构、反光或阴影，选 yes。如果没有这种误报，选 no；拿不准选 uncertain。
false_negative_region
如果 Raw 原图有明显动态区域，但 Probability/Soft mask 没覆盖或明显漏掉，选 yes。如果没有明显漏报，选 no；拿不准选 uncertain。
label_confidence
high：Raw 和辅助图都支持同一判断，很确定。
medium：基本能判断，但存在轻微遮挡、边界偏差或局部不清楚。
low：看不清、反光/模糊严重、目标太小或判断依据不足。
reviewer_notes
把不确定、遮挡、看不清、疑似反光、疑似阴影、边界偏移、overlay 明显误标/漏标等原因写在这里。写短句即可，例如“反光严重，无法确认是否为动态物体”。

常见情况

  • 明显移动/临时物体：dynamic_region_present 选 yes，dynamic_region_type 选 moving_object，label_confidence 选 high；如果 overlay 贴合，boundary_quality 可选 good。
  • overlay 只覆盖墙/地面/固定结构：dynamic_region_present 通常选 no，或 type 选 static_structure；false_positive_region 选 yes；boundary_quality 通常选 poor 或 not_applicable。
  • 看不清/反光/模糊：dynamic_region_present 选 uncertain，label_confidence 选 low，并在 reviewer_notes 写“反光强”“画面模糊”“遮挡严重”等原因。

导出后怎么做

页面会自动保存当前浏览器里的填写内容。完成全部样本后，点击底部“导出 CSV”；如果审计方要求 JSON，也可以在“导出 / 导入高级功能”里导出 JSON。把导出的文件发回给审计，不要自行修改字段名或删列。

移动端提示：默认单图大图查看，点图片可放大；图片加载失败时这里会显示具体 assets 路径。
