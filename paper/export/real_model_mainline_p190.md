## 2. Training Target Selection

### Candidate targets considered

1. **Object admission scorer (selected first)**
   - Learns the cluster-level decision `admit stable map object` vs `reject` from existing map-object evidence.
   - Uses already materialized TorWIC protocol outputs: selected/rejected clusters, session count, frame count, support, label purity, dynamic ratio, centroid/size features, category hints.
   - Requires no image-mask retraining, no SAM2 finetuning, and no long GPU run for the first smoke.

2. **Stable/dynamic object discriminator**
   - Useful, but narrower: it would mostly learn forklift/dynamic-agent rejection and not the full admission decision, including low-session / low-support infrastructure rejections.

3. **Cross-session association scorer**
   - Scientifically valuable, but pair/triplet labels are not yet materialized as a clean train/val/test table. It needs a separate pair-generation contract.

4. **Lightweight mask refinement**
   - Needs image/mask supervision and likely GPU/SAM2 integration. This is the wrong first step because the existing bottleneck is not mask quality but map admission / stable object selection.

### Selected first model target

**Cluster-level object admission scorer.**

Why this target first:

- It directly attacks the paper's core method rather than training a generic segmentation model.
- Labels already exist as weak supervision from the accepted/rejected cluster decisions in the current TorWIC protocol artifacts.
- It can run a CPU-only smoke test and establish dataset contracts before any heavier model training.
- It is the natural bridge from handcrafted boolean thresholds to a learned, auditable model.

## 4. Dataset Contract

Generated by:

```bash
python3 tools/prepare_admission_scorer_dataset_p190.py
```

Outputs:

- `paper/evidence/admission_scorer_dataset_p190.json`
- `paper/evidence/admission_scorer_dataset_p190.csv`

### Target label

`target_admit`:

- `1` = selected stable map object
- `0` = rejected cluster

Label source: existing TorWIC selected/rejected cluster artifacts from selection v5. This is weak supervision from the current rule-based system, not human relabeling.

### Features

- `session_count`
- `frame_count`
- `support_count`
- `dynamic_ratio`
- `label_purity`
- `mean_center_x`, `mean_center_y`
- `mean_size_x`, `mean_size_y`
- `is_forklift_like`
- `is_infrastructure_like`

### Split strategy

The split is protocol-aware rather than random:

- **Train:** same-day + cross-day Aisle (`21` samples)
- **Val:** cross-month Aisle (`14` samples)
- **Test:** Hallway within-site variation (`16` samples)

Total materialized samples: `51` cluster-level examples.

Label balance:

- Train: `10` admit / `11` reject
- Val: `7` admit / `7` reject
- Test: `9` admit / `7` reject

## 5. Baseline

First baseline is the existing transparent rule gate:

```text
session_count >= 2
frame_count >= 4
support_count >= 6
dynamic_ratio <= 0.2
label_purity >= 0.7
```

Smoke result on the weak-label dataset:

- Train accuracy: `1.000`
- Val accuracy: `1.000`
- Test accuracy: `1.000`

Interpretation: this is expected because the target labels are generated from the current rule policy. The purpose is not to claim model superiority yet. The purpose is to freeze a dataset contract and prepare the first learned scorer smoke. P191 must compare a learned model to this rule baseline and report whether it merely imitates the rule or exposes boundary cases.

## 6. Evaluation Metrics for Learned Scorer

Minimum metrics for P191:

- Accuracy by split
- Precision / recall / F1 for `target_admit=1`
- False admit rate for forklift-like clusters
- False reject rate for infrastructure-like clusters
- Calibration or score histogram if probabilistic model is used
- Feature importance / coefficient inspection for auditability

## 7. Minimal Training Script Gap

A dedicated training script does not yet exist. The smallest safe P191 implementation is:

```text
tools/train_admission_scorer_p191.py
```

Recommended model order:

1. Logistic regression or linear classifier on the P190 CSV table.
2. Decision tree / shallow random forest only if sklearn is available and the linear baseline is insufficient.
3. Keep the output auditable: save coefficients / tree rules and per-split predictions.

Do **not** start SAM2 finetuning as the first real-model experiment. SAM2 training requires image-mask supervision, checkpoint/config plumbing, and GPU memory, while the currently available labels most directly support cluster-level admission learning.

## 9. Immediate Risks

- Labels are weak labels from the current rule gate; a learned model can initially only imitate or smooth this decision boundary.
- Sample size is small (`51` clusters), so the first model must be treated as feasibility/smoke, not a publication-grade learned component.
- Cross-session association scorer remains a second-stage target after pair labels are materialized.
