\section{Introduction}
\label{sec:introduction}

% === I.A. Motivation ===
\subsection{Motivation}
\label{sec:motivation}

Simultaneous Localization and Mapping (SLAM) has matured to the point where short-horizon geometric maps can be reliably constructed in real time~\cite{cadena2016past}. The next frontier---long-term SLAM and map reuse across weeks or months---shifts the burden from geometric consistency to semantic persistence: which detected objects should enter the map, and which should be discarded as transient or dynamic contamination?

Industrial environments amplify this question. Warehouses and logistics centers contain stable infrastructure (barriers, work tables, racks, pillars) alongside mobile agents (forklifts, carts, human operators). An open-vocabulary detector such as Grounding DINO~\cite{liu2024grounding} combined with a segmenter such as SAM~2~\cite{ravi2024sam2} can propose bounding boxes and masks for dozens of object instances per frame, but without an explicit admission-control policy, the map quickly accumulates forklift-shaped phantom objects that are re-detected in revisits but should never have been admitted in the first place.

% === I.B. The Admission-Control Gap ===
\subsection{The Admission-Control Gap}
\label{sec:admission_gap}

Existing dynamic SLAM pipelines~\cite{bescos2018dynaslam} are designed to mask, track, and remove dynamic pixels and objects from the geometric reconstruction, ensuring that visual odometry and loop closure operate on clean static geometry. Object-level SLAM approaches~\cite{yang2019cubeslam,peng2023openscene,jatavallabhula2023conceptfusion} assign semantic identities to geometric entities and can reason about object categories. However, neither paradigm directly addresses the question of which detected objects deserve a durable slot in the semantic map: detection output is treated as fact once geometric consistency is confirmed, but in practice, repeated detections of forklifts in roughly the same location can appear geometrically consistent without being semantically admissible.

Our key observation is that the pipeline from open-vocabulary detection to durable semantic landmarks requires an explicit maintenance layer---an admission-control policy that scores each candidate object on stability (how long it persists across sessions), consistency (label agreement and spatial coherence), and dynamicity (whether its dominant state is \texttt{dynamic\_agent}). This paper describes such a layer.

% === I.C. Scope and Contribution Boundaries ===
\subsection{Scope and Contribution Boundaries}
\label{sec:scope}

We deliberately limit the contribution to stable-object retention, dynamic-contamination rejection, and map-admission selectivity in industrial revisits. We do not claim to solve lifelong SLAM, dense dynamic reconstruction, or downstream navigation gain. The paper offers an auditable bridge from perception to map maintenance, with reproducible evidence on the TorWIC dataset.

% II. CONTRIBUTIONS
% ============================================================================
\section{Contributions}
\label{sec:contributions}

\begin{enumerate}
  \item \textbf{An explicit object-maintenance architecture} (Section~\ref{sec:method}) with observation, tracklet, map-object, and revision layers that separates detection from map admission. Each layer carries frame-level provenance, stability counters, and dynamicity scores.
  
  \item \textbf{A transparent trust-score formulation} (Section~\ref{sec:trust_score}) for map admission that combines session-level stability, cross-session persistence, and dominant-state dynamicity. The score is intentionally simple and auditable rather than presented as an optimal estimator.
  
  \item \textbf{A reproducible evidence ladder} (Section~\ref{sec:results_aisle}) on the TorWIC dataset~\cite{barath2023povslam} across three Aisle protocols (same-day, cross-day, cross-month) and a Hallway broader-validation branch (Section~\ref{sec:results_hallway}), all reported with selection-criteria justification and cluster-ID-level traceability.
  
  \item \textbf{A rejection-profile analysis} (Section~\ref{sec:results_rejection}) showing that forklift-like dynamic contamination is the primary rejection driver (50.0\%--71.4\% share) across all four protocols.
  
  \item \textbf{An open-source release} of the object-maintenance layer with TorWIC protocol configuration and selection criteria, supporting auditable reproduction.
  
  \item \textbf{Admission-criteria defense} (Section~\ref{sec:results_defense}): parameter ablation confirming min\_sessions/min\_frames are sensitive filters and max\_dynamic\_ratio is naturally saturated by data bimodality; baseline comparison demonstrating that simpler heuristics (purity/support) cannot reject forklifts (phantom risk 21.1\% vs 0.0\% with the full policy).
  
  \item \textbf{Per-category retention and rejection analysis} (Section~\ref{sec:results_defense}): forklift rejection is universal (0/4 retained), infrastructure retention is selective and interpretable (40\%--50\%).
\end{enumerate}

\section{Related Work}
\label{sec:related_work}

% === III.A ===
\subsection{Semantic SLAM and Object-Level Mapping}
\label{sec:related_semantic}

Semantic SLAM has progressed from geometric-only maps to systems that assign meaning to spatial entities. Cadena et al.~\cite{cadena2016past} survey the SLAM landscape from geometric to semantic, establishing that map reuse requires semantic understanding. Early object-level approaches---CubeSLAM~\cite{yang2019cubeslam} (cuboids in monocular SLAM) and MaskFusion~\cite{runz2018maskfusion} (instance-level RGB-D tracking)---demonstrated that object identity improves robustness and interpretability. Kimera~\cite{rosinol2020kimera} provides a complete metric-semantic SLAM pipeline with mesh reconstruction and semantic labeling, while SemanticFusion~\cite{mccormac2017semanticfusion} enabled dense 3D semantic maps from CNN predictions.

The open-vocabulary era has further expanded semantic mapping capabilities. OpenScene~\cite{peng2023openscene} demonstrated that language-conditioned 3D feature fields carry concept-level semantics without closed-set training. ConceptFusion~\cite{jatavallabhula2023conceptfusion} fused multiple foundation model outputs into an open-set 3D map. LERF~\cite{kerr2023lerf} grounded CLIP embeddings within neural radiance fields, enabling open-vocabulary 3D queries. These systems share a common architectural assumption: once an object is detected, it is admissible to the map. Our work challenges this assumption by introducing an explicit admission-control policy informed by cross-session stability.

\textit{Relationship:} We build on semantic SLAM's object-level representations but focus on a complementary problem: which detected objects deserve permanent map representation? A forklift detected with 0.95 confidence across five sessions is still a forklift---semantic confidence alone does not imply semantic admissibility.

% === III.B ===
\subsection{Dynamic SLAM and Dynamic-Object Suppression}
\label{sec:related_dynamic}

Dynamic environments present a fundamental challenge for SLAM. DynaSLAM~\cite{bescos2018dynaslam} demonstrated that masking dynamic objects using Mask R-CNN and multi-view geometry improves camera tracking accuracy. Its follow-up, DynaSLAM~II~\cite{bescos2021dynaslam2}, added tightly-coupled multi-object tracking. Several systems target object-level dynamic handling: Co-Fusion~\cite{runz2017cofusion} segments and tracks multiple independently-moving objects, MID-Fusion~\cite{xu2019midfusion} uses an octree-based volumetric representation for multi-instance dynamic reconstruction, and FlowFusion~\cite{zhang2020flowfusion} leverages optical flow residuals for dynamic/static segmentation. DS-SLAM~\cite{yu2018dsslam} combines SegNet with a moving-consistency check, while Detect-SLAM~\cite{zhong2018detectslam} makes object detection and SLAM mutually beneficial by using detected keypoints only when objects are static. Saputra et al.~\cite{saputra2018visualslam} provide a comprehensive survey of visual SLAM in dynamic environments.

\textit{Relationship:} Dynamic SLAM removes dynamic content to protect geometric reconstruction (pose estimation and mapping). We address a different layer of the architecture: semantic map admission. An object may be correctly tracked as dynamic (DynaSLAM's goal) but still be erroneously admitted to the long-term semantic map (our concern). The two layers are complementary: dynamic SLAM cleans the geometric map, while our maintenance layer cleans the semantic map.

% === III.C ===
\subsection{Long-Term Map Maintenance}
\label{sec:related_longterm}

Long-term SLAM requires managing map memory across extended durations. RTAB-Map~\cite{labbe2019rtabmap} implements an appearance-based memory management system for large-scale and long-term operation. Churchill and Newman~\cite{churchill2012practice} demonstrated experience-based navigation where the robot maintains multiple map representations for different temporal conditions. Fehr et al.~\cite{fehr2016tsdf} proposed TSDF-based change detection for long-term dense reconstruction, distinguishing persistent structure from dynamic objects. ORB-SLAM3~\cite{campos2021orb3} introduced multi-map capabilities that enable map reuse and merging across sessions. The standard SLAM backends---ORB-SLAM2~\cite{murartal2015orb2} and DROID-SLAM~\cite{teed2021droidslam}---provide the visual odometry foundation on which our maintenance layer operates. POV-SLAM~\cite{barath2023povslam} introduced object-aware semantic SLAM and released the TorWIC dataset with ground-truth object annotations across multiple revisits.

\textit{Relationship:} Long-term mapping systems focus on memory management and geometric consistency across time. Our maintenance layer adds an orthogonal dimension: semantic admission control based on stability, persistence, and dynamicity signals---not just geometric or memory constraints.

% === III.D ===
\subsection{Segmentation-Assisted Filtering}
\label{sec:related_segmentation}

Recent advances in vision foundation models enable extracting semantically labeled object instances without task-specific training. Grounding~DINO~\cite{liu2024grounding} provides open-vocabulary detection by marrying DINO with grounded pre-training. SAM~2~\cite{ravi2024sam2} extends the Segment Anything paradigm to video with promptable spatio-temporal masks. OpenCLIP~\cite{cherti2023reproducible} provides reproducible CLIP-scale vision-language embeddings used for label reranking. At the 3D level, OpenMask3D~\cite{takmaz2023openmask3d} and CLIP-Fields~\cite{shafiullah2023clipfields} enable open-vocabulary 3D instance segmentation and semantic field learning, respectively---demonstrating that language-conditioned reasoning can operate directly on geometric representations.

\textit{Relationship:} We treat these models as interchangeable perception backends. Our maintenance layer consumes their output (detected object instances with semantic labels and masks) and applies session-level admission criteria. The architecture is backend-agnostic: any detection+segmentation pipeline producing frame-level object instances can feed into the maintenance layer.

\section{Limitations}
\label{sec:limitations}

\begin{enumerate}
  \item \textbf{Detection confidence.} Per-detection NN confidence scores were not logged during observation extraction. The label-purity proxy is used as a documented substitute.
  \item \textbf{Dataset scale.} TorWIC is the only publicly available multi-session industrial SLAM dataset with object annotations. Larger-scale field validation is deferred.
  \item \textbf{Single-session infrastructure loss.} Infrastructure appearing in only one session cannot be verified for cross-session persistence and is conservatively rejected.
  \item \textbf{Dynamic SLAM boundary.} Forklifts in TorWIC warehouse aisles occupy at most 1.39\% of the image frame, below the $\sim$5\% threshold needed to measurably affect DROID-SLAM. Richer dynamic scenarios may produce different outcomes.
  \item \textbf{VO backend.} Only DROID-SLAM was tested as the visual odometry backend. ORB-SLAM3 and other backends may have different sensitivity to dynamic content.
  \item \textbf{Citation completeness.} Current reference list (11 entries) requires expansion to meet TR-O expectations. Related-work expansion is planned (P164, see BUILD\_NOTES.md).
  \item \textbf{Ground-truth object annotations.} TorWIC provides ground-truth object IDs; we use them for protocol configuration but the framework itself operates without ground truth.
\end{enumerate}

\section{Conclusion}
\label{sec:conclusion}

This paper presents an object-centric approach to semantic-segmentation-assisted SLAM for dynamic industrial environments. The framework transforms open-vocabulary RGB-D segmentation outputs into observation, tracklet, map-object, and revision layers, then uses explicit persistence, stability, and dynamicity signals to retain stable semantic landmarks and suppress dynamic contamination.

Current TorWIC evidence demonstrates a reproducible submission-ready ladder across same-day, cross-day, and cross-month Aisle protocols, with Hallway retained as a separate scene-transfer branch. Forklift-like evidence is consistently rejected as dynamic contamination, while barriers, work tables, and warehouse racks are selectively retained based on multi-session evidence, label consistency, and static dominance.

The dynamic SLAM backend evaluation contributes a complete negative-result study: 10 DROID-SLAM configurations all produce trajectory-neutral results ($|\Delta\text{ATE}| < 0.1~\text{mm}$), with a quantified boundary condition at $\sim$5\% dynamic frame coverage. The admission criteria are validated through parameter ablation (confirming natural bimodality), baseline comparison (demonstrating that simpler heuristics fail), and per-category retention/rejection analysis (showing universal forklift rejection and interpretable infrastructure retention).

The contribution is bounded: not a lifelong SLAM benchmark, not dense dynamic reconstruction, and not a downstream navigation-gain claim, but a principled session-level map admission control methodology supported by auditable evidence.
